Aspire + Terraform Architecture

Use .NET Aspire for consistent local development and Terraform for production-ready cloud infrastructure.

Local Development

Use .NET Aspire to orchestrate PostgreSQL, Redis, MinIO, and MailHog locally with the same connection patterns you'll use in production.

Infrastructure as Code

Generated Terraform configurations manage all cloud resources with consistent, repeatable deployments across environments.

Multi-Cloud Ready

Same project structure works across AWS, Azure, Google Cloud, and container platforms with platform-specific Terraform modules.

AppHost Configuration Example

// Program.cs - AppHost for local orchestration
var builder = DistributedApplication.CreateBuilder(args);

// Local services (orchestrated by Aspire)
var postgres = builder.AddPostgres("postgres")
    .AddDatabase("myapi-db");

var redis = builder.AddRedis("redis");

var storage = builder.AddMinio("minio");

// API project with service connections
builder.AddProject<Projects.MyApi_Api>("myapi-api")
    .WithReference(postgres)
    .WithReference(redis)
    .WithReference(storage);

builder.Build().Run();
